# p5studio スケッチ生成スキル — ユーザーフィードバックから得た知見

ユーザーとの制作セッションで得られた「人間が求める品質」の傾向と、最初から期待通りの出力を出すためのガイドライン。

---

## 1. スピード感・テンポ

**傾向: 人間は「速い」方を好む。最初の提案は遅すぎることが多い。**

- アニメーションの初期値は「自分が思うよりテンポを速めに」設定する
- フレーム数のパラメータは控えめに（例: 接近アニメは6〜8フレーム程度で十分）
- 文字の出現間隔など「間」を詰めた方がリズム感が出る
- ユーザーが「もっと速く」と言ったら、半分〜2/3にするくらいで丁度いい

## 2. インパクト・大胆さ

**傾向: 控えめより大胆に振った方が好まれる。**

- サイズは画面からはみ出すくらいでちょうどいい（例: 1080pxキャンバスで文字サイズ1800）
- 傾き・回転は微妙にかけるより大きくかけた方が良い（±PIまで振ってOK）
- 「ランダム」を求められたら、おとなしいランダムではなく極端なランダムにする
- イーズアウトは `pow(4)` 以上のキレを使って「ドンッ」と止まる演出が効果的

## 3. 視点・構図の明確さ

**傾向: プロンプトの視点を正確に読み取ること。**

- 「海の波」→ 横から見た波か、真上から見た波か、プロンプトから読み取れない場合は確認する
- 自然現象（海、空、森など）は「真上」「斜め上」「横」で全く異なるコードになる
- 曖昧な場合はより印象的な視点（俯瞰・鳥瞰）をデフォルトにする

## 4. ちらつき・ノイズの制御

**傾向: 意図しないちらつきは即座に指摘される。品質の基本。**

- `random()` を毎フレーム呼ぶとちらつく → **Perlinノイズ（`noise()`）+ 時間軸** を使って滑らかに変化させる
- グレインエフェクト: `random()` ではなく `noise(px * scale, py * scale, frameCount * speed)` を使う
- コースティクス等のエフェクト: ランダム配置ではなくグリッドベースで位置を安定させ、ノイズで出現を制御
- ファイルサイズが異常に大きい場合（90MB超など）はちらつきの兆候

## 5. 色彩・鮮やかさ

**傾向: 最初は鮮やかさが足りない。彩度・明度を高めに。**

- 彩度（saturation）は75〜100の範囲を積極的に使う
- 明度（brightness）のコントラスト幅を広くとる（25〜70など）
- 色相の幅も広めにとると表情が豊かになる

## 6. アニメーションの終端

**傾向: 開始と終了の区切りをきちんと意識して設計する。**

- 「A〜Zを表示」のようなシーケンスは、最後の要素で正確に終わるように尺を設計する
- duration はコンテンツの長さから逆算して決める（余った黒フレームは不要）
- ループアニメの場合は `t = frameCount / TOTAL_FRAMES` で正規化して0→1で完結させる

## 7. エフェクトの強度

**傾向: エフェクトの初期提案は強すぎることが多い。**

- グレイン強度は20〜25程度が「フィルム感」として心地よい範囲
- 40以上は「荒い」印象になるので、ユーザーから「荒めに」と指定された場合のみ
- 半分にしてと言われたら素直に半分にする（数値で明確に）

## 8. 決定論的なランダム

**傾向: ランダム要素は `randomSeed()` で固定した上で使う。**

- スケジュール生成（タイミング、角度など）は `setup()` 内で `randomSeed()` → 事前計算
- `draw()` 内の `random()` はフレーム間の一貫性が保てないため最小限に
- ノイズベースのエフェクトは `noiseSeed()` で固定

- [ ] サイズ・回転は大胆か？（控えめになっていないか）
- [ ] `draw()` 内で `random()` を使っていないか？（ちらつきの原因）
- [ ] duration はコンテンツに合った長さか？（余白なし）
- [ ] 彩度・明度は十分か？（暗すぎないか）
- [ ] プロンプトの視点を正確に反映しているか？
